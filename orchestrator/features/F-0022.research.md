## Riesgos y restricciones técnicas para: Argos mobile — Posiciones Destacadas (top-5 plano con sparklines de área) en el Overview

**Resumen:** El top-5 plano con sparklines de área no existe actualmente; debe crearse nuevo. El componente Sparkline de área ya está disponible en @/components/ui y debe reemplazar al MiniSpark inline. El hook useSparklines provee los datos pero tiene restricciones de rate-limit, cacheo y filtrado de FCIs. No hay tests existentes para los componentes involucrados. El layout actual (filas de 56px con MiniSpark de 20px) debe ajustarse para alojar el área sparkline de 40px. La prop sparklineFor debe sincronizarse con el nuevo componente y no se pudo verificar su origen exacto (marcado NO VERIFICADO).

### Riesgos / Restricciones
- El hook useSparklines filtra tickers FCI (no tienen histórico) y usa 400ms staggered requests con límite de 120 req/min — puede causar latencia si los top-5 tienen muchos tickers no cacheados
- El componente Sparkline (área) tiene 40px de alto por defecto; el MiniSpark inline actual usa 48x20px — hay que ajustar layout para que no rompa el diseño actual de filas de 56px min-height
- El sparklineFor prop se pasa desde DashboardOverview pero no se encontró dónde se construye el mapa ticker→data (NO VERIFICADO) — posiblemente en el padre del Overview
- MobilePositionsList ya ordena por valuation descendente; un top-5 plano debe sincronizarse con ese orden o definirse un criterio de ranking distinto (ej. by resultPct) y comunicarlo claramente al usuario
- No se encontraron tests existentes para MobilePositionsList, Sparkline ni DashboardOverview — cambios pueden romperse sin detección automática
- Posiciones con menos de 2 puntos de historia causan que Sparkline retorne null (verificado en Sparkline.jsx y useSparklines.js) — el top-5 debe manejar ese caso sin huecos visuales
- La data912 cachea en localStorage con TTL 24h — el top-5 puede mostrar sparklines desactualizados hasta nuevo fetch
- Los estilos usan tokens CSS custom (--surface-2, --border, --ink, --teal, etc.) y la variante mobile usa clases md:hidden/hidden md:block — el top-5 debe respetar ese patrón responsive
