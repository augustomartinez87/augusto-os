## Contexto

Depende de F-0013 a F-0016 y de ConfigManoDeObra/GastoOperativo (bootstrap). Si todavía no hay gastos cargados en el período, costoOperativo = 0 con una nota visible de "sin gastos cargados este período" — no debe romper el cálculo.

## Pasos sugeridos

1. Queries agregadas por período (hoy/semana/mes) sobre Venta, Lote, GastoOperativo.
2. Cálculo de "empanadas producibles con stock actual": mín, para cada insumo de la receta, de `stockActual_insumo / cantidadPorUnidad_insumo`.
3. UI del dashboard.
4. Tests de las fórmulas de agregación y del cálculo de costo.

## Fuera de alcance

- Gráficos comparativos entre meses (roadmap post-MVP, `SPEC-MVP.md` §9).
- Exportar reporte (PDF/Excel).

## Restricciones clave

- Nunca mostrar "costos fijos/variables" en la UI — usar el lenguaje de `SPEC-MVP.md` §6.
- El patrón de ventas mostrado es siempre el real registrado, nunca una proyección basada en estimaciones verbales de Dani (ver `SPEC-MVP.md` §8.1).
- Sin migración de schema.
