## Contexto

Primer módulo transaccional. Sin esto no hay costo de materia prima real (ver `SPEC-MVP.md` §5 en la raíz del repo tres-saltenas). Requiere que el bootstrap (Fase A, fuera del loop) ya haya materializado el schema `Insumo`/`Compra` en Neon dev — si no existe, este feature está bloqueado, no lo ejecutes.

## Pasos sugeridos

1. Router tRPC `compra.create` (input: insumoId, cantidad, precioTotal, proveedor, fecha) — transacción Prisma que crea el registro de Compra y actualiza el Insumo (costoUnitarioActual, stockActual += cantidad).
2. Página/formulario de carga, mobile-first (Dani lo usa desde el celular).
3. `compra.list` + UI de historial.
4. Tests unitarios del cálculo de costoUnitarioActual (no e2e).

## Fuera de alcance

- Multi-proveedor o comparación de precios entre proveedores.
- Edición o borrado de compras ya cargadas.
- Adjuntar foto de factura/comprobante.

## Restricciones clave

- Lenguaje de UI: "Compras", nunca "Egresos" ni jerga contable.
- costoUnitarioActual siempre usa el último precio cargado (no promedio ponderado histórico).
- Sin migración de schema — ya existe desde el bootstrap.
