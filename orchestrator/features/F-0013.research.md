## Riesgos y restricciones técnicas para: 3 Salteñas — Módulo Compras

**Resumen:** El Módulo Compras está completamente por construir: tanto el router tRPC (compraRouter) como la página UI (compras/page.tsx) son placeholders vacíos. El riesgo principal es el coupling fuerte entre Compra e Insumo: cada compra debe actualizar costoUnitarioActual y stockActual del insumo en una misma transacción Prisma, y de eso depende todo el costeo downstream (Lotes, Dashboard). Además, no existe configuración de cliente tRPC (src/trpc está vacía), no hay esquemas Zod de validación ni tests unitarios para la lógica de negocio. Se debe implementar con cuidado la transacción atómica Compra→Insumo, usando publicProcedure (o migrar a protectedProcedure con Clerk) y superjson como transformer.
